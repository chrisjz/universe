# Universe Atlas — working notes for AI assistants

A true-scale WebGPU atlas of the universe. Live at https://universeatlas.org;
data tiles served from https://data.universeatlas.org (repo `chrisjz/universe-data`).
Read `docs/ARCHITECTURE.md` before touching the renderer or scene.

## Commands

- `npm run dev` — Vite dev server. `npm run build` — typecheck + bundle.
- `npm run lint` / `npm run format:check` — both are CI gates; a pre-commit
  hook auto-formats staged files.
- `node scripts/capture-views.mjs [dir]` — screenshot the 8 regression views
  from a local headless Chrome. `node scripts/compare-views.mjs [dir]` —
  pixel-diff against `tests/visual/baseline/`.
- `node scripts/verify-ephemeris.mjs` — check the planet ephemeris against
  live JPL Horizons (<0.2° asserted).

## Hard conventions

- **Scene coordinates**: ecliptic longitude λ → x=cos λ, z=−sin λ, y=north.
  Ecliptic (x,y,z) maps to scene (x, z, −y). All positions are camera-relative
  in uniforms; distances in meters. J2000 is the epoch for everything.
- **True scale is the product.** Never fudge a size, distance, or rate for
  looks. Every real dataset gets a credit in README's Data & credits section.
- **Dynamic uniform slots are 256 B** (`SLOT`); bind-group binding sizes in
  `mkDyn(...)` must match the WGSL struct size exactly — an undersized
  binding invalidates the whole render pass _silently_ (black screen, no
  errors).
- Deterministic URL params exist for testing: `?at=<ISO>` (sim epoch),
  `?paused=1`, `?norender=1` (never submit a frame), `?stars=athyg` (bundled
  catalog only), `?goto=<slug>&dist=&yaw=&pitch=`, `?fps=1` (fps in title).

## Verification

- Verify rendering changes headlessly before pushing: capture-views locally,
  or drive Chrome with `--headless=new --enable-unsafe-webgpu` (real GPU).
  Blank frames under `--virtual-time-budget` are a dev-server artifact, not
  a product bug — retry or use real-time puppeteer.
- CI renders the real app on Mesa lavapipe (software Vulkan). Two rules make
  that work: CI never presents a frame (`?norender=1`) and pixels are read
  back via `renderer.snapshot()` / `window.__snap()`, never via screenshots
  or `canvas.toBlob` (those read black on software stacks).
- Visual baselines are **CI pixels, not local pixels**: to refresh after an
  intended visual change, download the `visual` artifact from the PR's
  Actions run, copy the PNGs into `tests/visual/baseline/`, commit.

## Process

- One focused PR at a time, always based on `main` — no stacked PRs. The
  maintainer merges each before the next starts.
- Commit messages: evocative first line, then a short story of the why.
  Multi-line messages via `git commit -F <file>`.
- `gh pr edit` is broken by a GraphQL deprecation — use
  `gh api -X PATCH repos/chrisjz/universe/pulls/<n>` instead.
- Prettier reflows README tables and long lines: after editing README.md or
  index.html with exact-string replaces, grep-verify the change actually
  landed.
- Data files in `public/` are regenerated by `scripts/generate-*.mjs` from
  public sources — never hand-edit binaries; rerun the generator.
